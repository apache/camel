The Batch component provides structured batch processing for collections of items.
It handles chunking, error thresholds, watermark-based resume, and progress reporting,
replacing the need for manual split + aggregate + error handling assembly.

This fills a gap similar to MuleSoft's `<batch:job>` which Camel users previously achieved by
manually combining the Splitter EIP, Aggregator EIP, and custom error handling logic.

== URI format

----
batch:jobName[?options]
----

Where `jobName` is a unique name identifying the batch job.

== Usage

You send a collection (`List`, `Iterator`, or any body convertible to `Iterator`) to the batch endpoint.
The component processes items in configurable chunks with error handling and optional watermark tracking.

=== Basic Example

[tabs]
====
Java::
+
[source,java]
----
from("direct:start")
    .to("batch:myJob?chunkSize=100&processorRef=direct:processItem");

from("direct:processItem")
    .log("Processing item: ${body}")
    .to("mock:result");
----
====

=== Error Threshold

The `errorThreshold` option controls when a batch job should be aborted based on the ratio of failures.
The value is a fraction between 0.0 and 1.0:

* `1.0` (default) -- never abort, process all items regardless of failures
* `0.1` -- abort if more than 10% of items fail
* `0.0` -- abort on the first failure

[tabs]
====
Java::
+
[source,java]
----
from("direct:start")
    .doTry()
        .to("batch:importJob?chunkSize=50&processorRef=direct:import&errorThreshold=0.1")
    .doCatch(BatchException.class)
        .log("Batch aborted: ${exception.message}")
    .end();
----
====

When the threshold is exceeded, a `BatchException` is thrown containing the `BatchResult` with
full statistics about the batch execution.

=== Watermark Tracking

Watermark tracking enables resume-from-last-position for batch jobs. By providing a watermark store
(any `Map<String, String>` bean), the component tracks how many items have been processed and
skips already-processed items on subsequent runs.

[tabs]
====
Java::
+
[source,java]
----
from("timer:poll?period=60000")
    .bean("myDataSource", "fetchAllRecords")
    .to("batch:syncJob?chunkSize=100&processorRef=direct:sync&watermarkStore=#myWatermarkStore");
----
====

The watermark key defaults to the job name but can be customized with `watermarkKey`.

=== Parallel Processing

Enable `parallelProcessing` to process items within each chunk concurrently:

[tabs]
====
Java::
+
[source,java]
----
from("direct:start")
    .to("batch:fastJob?chunkSize=50&processorRef=direct:process&parallelProcessing=true");
----
====

=== Aggregation Strategy

An optional `AggregationStrategy` can be provided to collect results from processed items:

[tabs]
====
Java::
+
[source,java]
----
from("direct:start")
    .to("batch:transformJob?chunkSize=100&processorRef=direct:transform&aggregationStrategy=#myStrategy");
----
====

== Batch Result

After processing, the exchange body is set to a `BatchResult` object (unless an aggregation strategy
is configured) containing:

* `totalItems` -- total number of items processed
* `successCount` -- number of successfully processed items
* `failureCount` -- number of failed items
* `duration` -- processing time in milliseconds
* `aborted` -- whether the batch was aborted
* `failures` -- list of `BatchFailure` objects with index, item, and cause

== Output Headers

[width="100%",cols="25%,15%,60%",options="header"]
|===
| Header | Type | Description
| CamelBatchJobName | String | The batch job name
| CamelBatchTotal | int | Total items processed
| CamelBatchSuccess | int | Successfully processed items
| CamelBatchFailed | int | Failed items
| CamelBatchDuration | long | Total processing time in milliseconds
| CamelBatchAborted | boolean | Whether the batch was aborted due to error threshold
|===

== Item Exchange Headers

Each item exchange sent to the processor endpoint receives these headers:

[width="100%",cols="25%,15%,60%",options="header"]
|===
| Header | Type | Description
| CamelBatchJobName | String | The batch job name
| CamelBatchIndex | int | Zero-based index of this item
| CamelBatchSize | int | Total number of items in the batch
| CamelBatchChunkIndex | int | Index of the current chunk
|===
