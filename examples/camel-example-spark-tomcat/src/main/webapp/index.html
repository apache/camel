Spark and Apache Tomcat example

This example shows how to use Spark Java to define REST endpoints in Camel routes using the CamelSparkRouteBuilder DSL
The routes are defined in Java code, in the org.apache.camel.example.spark.MySparkRouteBuilder class.

There is a hello REST service that supports different content-type

  • text/plain
  • text/xml
  • application/xml
The REST service is defined using Spark syntax as hello/:me which means that it starts with a fixed hello/ path, and then accepts a parameter which is named :me which will be mapped to a Camel header with key me.
This allows you to access the REST service and include a parameter, for example

hello/world

This means that world is the parameter.

You can access the text/plain service from the web browser by clicking this link.

You can try the text/xml or application/json from the command line using curl or similar tool. 

    curl -H "Accept: application/json" http://localhost:8080/camel-example-spark-tomcat/hello/you
    curl -H "Accept: text/xml" http://localhost:8080/camel-example-spark-tomcat/hello/you
This assume you installed the example by copying the .war as camel-example-spark-tomcat.war into the webapps directory of Apache Tomcat.
